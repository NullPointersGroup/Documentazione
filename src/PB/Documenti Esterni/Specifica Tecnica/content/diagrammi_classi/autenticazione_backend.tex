\subsubsubsection{UserRepoPort}
\begin{figure}[H]
	\centering 
	\includegraphics[width=0.6\textwidth]{SpecificaTecnica/Autenticazione/UserRepoPort}
	\caption{Autenticazione e Registrazione: UserRepoPort}
\end{figure}

\paragraph{Descrizione}
\texttt{UserRepoPort} è un'interfaccia che definisce il contratto per l'accesso ai dati degli utenti.\\
Viene implementata da \texttt{UserRepoAdapter} e utilizzata dai service del dominio, garantendo il disaccoppiamento tra logica di business e layer di persistenza.

\paragraph{Attributi}
\texttt{UserRepoPort} non definisce attributi, in quanto si tratta di un'interfaccia.

\paragraph{Metodi e funzioni}
\begin{itemize}
	\item \texttt{find\_by\_username(username: string): Utente} - metodo pubblico che recupera un utente dal repository a partire dal suo username.\\
	Restituisce l'oggetto \texttt{Utente} corrispondente.
	\item \texttt{username\_exists(username: string): bool} - metodo pubblico che verifica se un determinato username è già presente nel sistema.\\
	Restituisce \texttt{true} se l'username esiste, \texttt{false} altrimenti.
	\item \texttt{email\_exists(email: string): bool} - metodo pubblico che verifica se un determinato indirizzo email è già registrato nel sistema.\\
	Restituisce \texttt{true} se l'email esiste, \texttt{false} altrimenti.
	\item \texttt{add\_user(u: UserRegistration): bool} - metodo pubblico che aggiunge un nuovo utente al repository a partire dai dati di registrazione forniti.\\
	Restituisce \texttt{true} in caso di successo.
	\item \texttt{delete\_user(username: string)} - metodo pubblico che rimuove dal repository l'utente identificato dall'username fornito.
	\item \texttt{reset\_password(u: UserReset)} - metodo pubblico che aggiorna la password dell'utente utilizzando i dati contenuti nell'oggetto \texttt{UserReset}.
\end{itemize}


\subsubsubsection{UserRepoAdapter}
\begin{figure}[H]
	\centering 
	\includegraphics[width=0.5\textwidth]{SpecificaTecnica/Autenticazione/UserRepoAdapter}
	\caption{Autenticazione e Registrazione: UserRepoAdapter}
\end{figure}

\paragraph{Descrizione}
\texttt{UserRepoAdapter} è la classe concreta che implementa l'interfaccia \texttt{UserRepoPort}, fungendo da adattatore tra il dominio applicativo e il layer di persistenza.\\
Delega le operazioni effettive sul database all'istanza di \texttt{UserRepository} che incapsula al proprio interno.

\paragraph{Attributi}
\begin{itemize}
	\item \texttt{repo: UserRepository} - attributo privato di tipo \texttt{UserRepository}.\\
	Rappresenta il repository concreto utilizzato per eseguire le query sul database.
\end{itemize}

\paragraph{Metodi e funzioni}
\begin{itemize}
	\item \texttt{\_\_init\_\_(db: Session)} - metodo pubblico di inizializzazione.\\
	Riceve una sessione di database di tipo \texttt{Session} e la utilizza per istanziare il \texttt{UserRepository} interno.
\end{itemize}


\subsubsubsection{UserRepository}
\begin{figure}[H]
	\centering 
	\includegraphics[width=0.6\textwidth]{SpecificaTecnica/Autenticazione/UserRepository}
	\caption{Autenticazione e Registrazione: UserRepository}
\end{figure}

\paragraph{Descrizione}
\texttt{UserRepository} è la classe che gestisce direttamente le operazioni di accesso e manipolazione dei dati degli utenti sul database.\\
Utilizza un \texttt{QueryExecutor} interno per eseguire le query in modo centralizzato e uniforme.

\paragraph{Attributi}
\begin{itemize}
	\item \texttt{executor: QueryExecutor} - attributo privato di tipo \texttt{QueryExecutor}.
	Componente utilizzato per l'esecuzione delle query sul database, centralizzando la gestione delle operazioni di persistenza.
\end{itemize}

\paragraph{Metodi e funzioni}
\begin{itemize}
	\item \texttt{\_\_init\_\_(db: Session)} - metodo pubblico di inizializzazione.
	Riceve una sessione di database e la utilizza per configurare il \texttt{QueryExecutor}.
	\item \texttt{find\_by\_username(username: string): Utente} - metodo pubblico che esegue una query per recuperare l'utente corrispondente all'username fornito.\\
	Restituisce l'oggetto \texttt{Utente} trovato.
	\item \texttt{find\_by\_email(email: string): Utente} - metodo pubblico che esegue una query per recuperare l'utente corrispondente all'indirizzo email fornito.\\
	Restituisce l'oggetto \texttt{Utente} trovato.
	\item \texttt{save(u: UserRegistration): bool} - metodo pubblico che persiste un nuovo utente nel database a partire dall'oggetto \texttt{UserRegistration} fornito.\\
	Restituisce \texttt{true} in caso di successo.
	\item \texttt{delete(username: string): bool} - metodo pubblico che rimuove dal database l'utente identificato dall'username fornito.\\
	Restituisce \texttt{true} se l'operazione va a buon fine.
	\item \texttt{reset\_password(u: UserReset): bool} - metodo pubblico che aggiorna la password dell'utente nel database utilizzando i dati dell'oggetto \texttt{UserReset}.\\
	Restituisce \texttt{true} in caso di successo.
\end{itemize}


\subsubsubsection{CheckUserService}
\begin{figure}[H]
	\centering 
	\includegraphics[width=0.6\textwidth]{SpecificaTecnica/Autenticazione/CheckUserService}
	\caption{Autenticazione e Registrazione: CheckUserService}
\end{figure}

\paragraph{Descrizione}
\texttt{CheckUserService} è il service applicativo responsabile della verifica delle credenziali durante il processo di login.\\
Interagisce con il repository tramite la porta \texttt{UserRepoPort} per recuperare i dati dell'utente e verificarne la password.

\paragraph{Attributi}
\begin{itemize}
	\item \texttt{port: UserRepoPort} - attributo privato di tipo \texttt{UserRepoPort}.
	Porta utilizzata per accedere ai dati degli utenti nel repository.
\end{itemize}

\paragraph{Metodi e funzioni}
\begin{itemize}
	\item \texttt{\_\_init\_\_(port: UserRepoPort)} - metodo pubblico di inizializzazione.
	Riceve e memorizza la porta di accesso al repository degli utenti.
	\item \texttt{check\_user(u: User): string} - metodo pubblico che verifica l'esistenza dell'utente e la correttezza della password fornita.\\
	Restituisce un token di autenticazione in caso di successo, oppure un messaggio di errore in caso di fallimento.
	\item \texttt{\_verify\_password(plain: str, hashed: str): bool} - metodo privato che confronta una password in chiaro con la corrispondente versione hashata.\\
	Restituisce \texttt{true} se le due stringhe corrispondono.
\end{itemize}

\subsubsubsection{RegisterUserService}
\begin{figure}[H]
	\centering 
	\includegraphics[width=0.8\textwidth]{SpecificaTecnica/Autenticazione/RegisterUserService}
	\caption{Autenticazione e Registrazione: RegisterUserService}
\end{figure}

\paragraph{Descrizione}
\texttt{RegisterUserService} è il service applicativo responsabile della registrazione di nuovi utenti.\\
Verifica la disponibilità di username ed email, valida il dominio dell'indirizzo email tramite \texttt{EmailValidationPort} e persiste il nuovo utente tramite \texttt{UserRepoPort} previa cifratura della password.

\paragraph{Attributi}
\begin{itemize}
	\item \texttt{port: UserRepoPort} - attributo privato di tipo \texttt{UserRepoPort}.\\
	Porta utilizzata per accedere al repository degli utenti e verificare la disponibilità delle credenziali e salvare il nuovo utente.

	\item \texttt{email\_validator: EmailValidationPort} - attributo pubblico di tipo \texttt{EmailValidationPort}.\\
	Porta utilizzata per validare l'esistenza del dominio dell'indirizzo email fornito durante la registrazione.
\end{itemize}

\paragraph{Metodi e funzioni}
\begin{itemize}
	\item \texttt{\_\_init\_\_(repo: IUserRepoPort, email\_validator: EmailValidationPort)} - metodo pubblico di inizializzazione.\\
	Riceve e memorizza la porta del repository e il validatore email.

	\item \texttt{register\_user(u: UserRegistration)} - metodo pubblico che gestisce il flusso completo di registrazione: verifica la disponibilità di username ed email, valida il dominio email, cifra la password e salva il nuovo utente tramite il repository.

	\item \texttt{\_hash\_password(password: string): string} - metodo privato che cifra la password in chiaro e restituisce la corrispondente stringa hashata da memorizzare nel database.
\end{itemize}

\subsubsubsection{ResetPasswordService}
\begin{figure}[H]
	\centering 
	\includegraphics[width=0.6\textwidth]{SpecificaTecnica/Autenticazione/ResetPasswordService}
	\caption{Autenticazione e Registrazione: ResetPasswordService}
\end{figure}

\paragraph{Descrizione}
\texttt{ResetPasswordService} è il service applicativo responsabile del reset della password di un utente esistente.\\
Verifica la correttezza della password attuale, cifra la nuova password e delega l'aggiornamento al repository tramite \texttt{UserRepoPort}.

\paragraph{Attributi}
\begin{itemize}
	\item \texttt{port: UserRepoPort} - attributo privato di tipo \texttt{UserRepoPort}.\\
	Porta utilizzata per accedere al repository degli utenti e delegare l'operazione di aggiornamento della password.
\end{itemize}

\paragraph{Metodi e funzioni}
\begin{itemize}
	\item \texttt{\_\_init\_\_(port: UserRepoPort)} - metodo pubblico di inizializzazione.\\
	Riceve e memorizza la porta di accesso al repository degli utenti.

	\item \texttt{reset\_password(u: UserReset)} - metodo pubblico che gestisce il flusso di reset della password: verifica la password attuale, cifra quella nuova e aggiorna il record nel repository.

	\item \texttt{\_hash\_password(password: string): string} - metodo privato che cifra la nuova password in chiaro e restituisce la corrispondente stringa hashata.

	\item \texttt{\_verify\_password(plain: string, hashed: string): bool} - metodo privato che confronta la password in chiaro fornita dall'utente con quella hashata presente nel database.\\
	Restituisce \texttt{true} se le due stringhe corrispondono.
\end{itemize}
