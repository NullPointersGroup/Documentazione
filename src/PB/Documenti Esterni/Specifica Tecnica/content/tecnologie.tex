\section{Tecnologie}

In questa sezione vengono descritte le tecnologie adottate per lo sviluppo del progetto \textbf{SmartOrder}, suddivise per ambito di utilizzo. Per ciascuna tecnologia vengono riportate la versione di riferimento, la documentazione ufficiale e il ruolo ricoperto all'interno del sistema.

\subsection{Linguaggi di programmazione}

\subsubsection{Python}
\begin{itemize}
    \item \textbf{Versione:} 3.14.3
    \item \textbf{Documentazione:} \url{https://docs.python.org/3/}
\end{itemize}

Python$^G$ è un linguaggio di programmazione ad alto livello, interpretato e multiparadigma, progettato per favorire la leggibilità del codice e la rapidità nello sviluppo delle applicazioni.

Nel progetto \textbf{SmartOrder} viene utilizzato principalmente per lo sviluppo del backend applicativo e per l'integrazione con i modelli di intelligenza artificiale$^G$. In particolare, gestisce la logica di elaborazione degli ordini, l'implementazione delle API$^G$ tramite FastAPI$^G$ e l'interazione con il database$^G$ e con i sistemi di memorizzazione vettoriale. Grazie al suo ampio ecosistema di librerie, Python$^G$ risulta particolarmente adatto allo sviluppo di applicazioni basate su tecniche di machine learning$^G$ e intelligenza artificiale$^G$.

\subsubsection{TypeScript}
\begin{itemize}
    \item \textbf{Versione:} 5.9
    \item \textbf{Documentazione:} \url{https://www.typescriptlang.org/docs/}
\end{itemize}


TypeScript è un linguaggio di programmazione open source che estende JavaScript introducendo la tipizzazione statica, interfacce, tipi complessi e generics. I controlli sui tipi a compile-time riducono gli errori comuni legati all'assenza di tipizzazione e migliorano la leggibilità, la manutenibilità e la robustezza del codice.
Nel progetto$^G$ viene utilizzato per gestire dati strutturati complessi come ordini, prodotti, quantità e validazioni, contribuendo a mantenere coerenza tra frontend$^G$ e backend$^G$ e a ridurre il rischio di errori nella comunicazione tramite API.


\subsection{Frontend}

\subsubsection{React}
\begin{itemize}
    \item \textbf{Versione:} 19.2
    \item \textbf{Documentazione:} \url{https://it.react.dev/reference/react}
\end{itemize}

React$^G$ è una libreria JavaScript open source per la costruzione di interfacce utente basate su un modello dichiarativo e su componenti indipendenti e riutilizzabili. Utilizza un DOM virtuale per aggiornare selettivamente solo le parti dell'interfaccia effettivamente modificate, ottimizzando le prestazioni di rendering.

Non essendo opinionata nella scelta delle librerie di routing o di gestione dello stato, non impone vincoli architetturali, permettendo di strutturare il frontend in modo modulare e adattarlo alle esigenze dell'applicazione.

\subsubsection{TailwindCSS}
\begin{itemize}
    \item \textbf{Versione:} 4.2
    \item \textbf{Documentazione:} \url{https://tailwindcss.com/docs/installation/using-vite}

TailwindCSS$^G$ è un framework$^G$ CSS utility-first che mette a disposizione un ampio insieme di classi di utilità che applicano singole proprietà CSS direttamente nel markup. Questo approccio consente di costruire interfacce in modo rapido e coerente, evitando la scrittura di fogli di stile personalizzati e complessi, e favorendo una maggiore manutenibilità$^G$ del codice di presentazione.

\subsubsection{Vite}
\begin{itemize}
    \item \textbf{Versione:} 7.3.1
    \item \textbf{Documentazione:} \url{https://vite.dev/guide/#scaffolding-your-first-vite-project}
\end{itemize}

Vite$^G$ è uno strumento di build per lo sviluppo di applicazioni web moderne. Sfrutta i moduli ES nativi del browser per avviare l'ambiente di sviluppo con tempi di avvio ridotti, offrendo hot module replacement per aggiornamenti immediati durante lo sviluppo. Supporta la configurazione tramite plugin, l'ottimizzazione automatica delle dipendenze e la generazione di bundle di codice ottimizzati tramite Rollup$^G$.

\subsection{Backend}

\subsubsection{FastAPI}
\begin{itemize}
    \item \textbf{Versione:}
    \item \textbf{Documentazione:} \url{https://fastapi.tiangolo.com}
\end{itemize}

FastAPI$^G$ è un framework$^G$ web per Python$^G$ utilizzato per sviluppare API$^G$ REST$^G$ ad alte prestazioni. Si basa sullo standard ASGI$^G$ e utilizza librerie come Pydantic$^G$ e Starlette$^G$ per la gestione delle richieste e la validazione$^G$ dei dati.

Nel progetto \textbf{SmartOrder} viene utilizzato per implementare il backend dell'applicazione e gestire la comunicazione tra frontend, moduli di elaborazione e database$^G$. Inoltre, consente di integrare facilmente i modelli di intelligenza artificiale$^G$ e di generare automaticamente la documentazione delle API$^G$ tramite strumenti basati su OpenAPI$^G$.


\subsubsection{LangChain}
\begin{itemize}
    \item \textbf{Versione:}
    \item \textbf{Documentazione:} \url{https://python.langchain.com}
\end{itemize}

LangChain$^G$ è un framework$^G$ open source progettato per facilitare lo sviluppo di applicazioni basate su Large Language Models$^G$ (LLM$^G$). Fornisce strumenti per gestire prompt, orchestrare le chiamate ai modelli e integrare i modelli di linguaggio con fonti di dati esterne.

Nel progetto \textbf{SmartOrder} viene utilizzato per gestire l'interazione con il modello di linguaggio e costruire le pipeline di elaborazione necessarie per interpretare gli input degli utenti ed estrarre le informazioni rilevanti dagli ordini, trasformandole in dati strutturati utilizzabili dal sistema.

\subsubsection{FAISS}
\begin{itemize}
    \item \textbf{Versione:}
    \item \textbf{Documentazione:} \url{https://faiss.ai/index.html}
\end{itemize}

FAISS$^G$ (Facebook AI Similarity Search) è una libreria open source sviluppata da Meta$^G$ per l'indicizzazione e la ricerca efficiente di vettori ad alta dimensionalità, utilizzata principalmente per operazioni di similarity search tra embedding$^G$.

Nel progetto \textbf{SmartOrder} viene utilizzato come database vettoriale per memorizzare e confrontare gli embedding$^G$ generati dai modelli di linguaggio. Questo consente di individuare prodotti simili a partire da descrizioni testuali e migliorare l'accuratezza del matching tra le richieste dell'utente e il catalogo aziendale.

\subsection{Database}

\subsubsection{PostgreSQL}
\begin{itemize}
    \item \textbf{Versione:}
    \item \textbf{Documentazione:} \url{https://www.postgresql.org/docs/}
\end{itemize}

PostgreSQL$^G$ è un sistema di gestione di basi di dati relazionali open source conforme agli standard SQL$^G$, noto per la sua affidabilità e robustezza nella gestione dei dati.

Nel progetto \textbf{SmartOrder} viene utilizzato come database$^G$ principale per la memorizzazione persistente delle informazioni dell'applicazione, tra cui il catalogo dei prodotti, gli ordini generati dal sistema e i dati associati agli input elaborati. L'utilizzo di PostgreSQL$^G$ consente di garantire integrità, consistenza e affidabilità nella gestione dei dati e si integra facilmente con il backend sviluppato in Python$^G$.
