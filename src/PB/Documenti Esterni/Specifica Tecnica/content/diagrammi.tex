\section{Diagrammi delle classi}

Dato che nelle tecnologie React e FastAPI le classi non sono una convenzione delle tecnologie, queste nel diagramma vengono mostrate come classi ma nella \vr{Descrizione} di ogni singola classe viene specificato il loro utilizzo.\\
Nelle classi di tipo frontend verranno omessi volutamente gli attributi, in quanto non sono rilevanti nella scrittura del presente documento.\\
Ad esempio, nel caso di un componente React, verrà scritto nella descrizione che quella classe è da intendersi come componente.\\
Ad esempio, nel caso di endpoint FastAPI, verrà scritto nella descrizione che quella classe  è responsabile delle richieste HTTP e, più in generale, delle operazioni di tipo API REST.

\subsection{Autenticazione e Registrazione}

Questa parte si occupa di ricevere le richieste di ottenimento Token$^G$, controllarne i valori e restituire un Token$^G$ valido e temporaneo (24 ore di validità), affinché il Client possa utilizzare il Sistema.\\
I Token$^G$ sono firmati tramite l'algoritmo \textbf{HS256} (HMAC-SHA256), che utilizza una chiave segreta condivisa sia per la firma che per la verifica.\\
Le password degli utenti sono invece cifrate tramite \textbf{bcrypt}, algoritmo appositamente progettato per l'hashing.
